# Habitify-Inspired App Docs

This folder is the source of truth for the HabitFlow product direction.
The goal is a Habitify-inspired habit tracking app, not a brand or UI copy.

## Docs

- [Product PRD](./product-prd.md): product vision, user problems, feature scope, success metrics, and build phases.
- [Frontend Expo React Native Spec](./frontend-expo-react-native-spec.md): mobile app architecture, screens, flows, state, navigation, and client behavior.
- [Backend PRD](./backend-prd.md): backend domain model, API surface, services, jobs, integrations, and system risks.
- [Design System UI](./design-system-ui.md): visual direction, tokens, components, interaction rules, and screen composition.

## Senior Build Note

The hardest parts are not the checklist UI. The hard parts are recurrence,
timezone-safe habit logs, notification reliability, analytics integrity, and
sync conflict handling across devices.
